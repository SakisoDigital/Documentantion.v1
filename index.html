Getting Started

Welcome to Sakiso Stream Documentation! We would like to thank you for choosing us!.

The School Management System is built with the latest versions of React and Node.js , offering a modern and user-friendly platform for automating school administrative and academic tasks. It features a responsive design, modular components, and a scalable backend, ensuring seamless interaction for administrators, teachers, students, and parents. The system supports both single-page and multi-page layouts, providing flexibility and easy customization for educational institutions.

Requirements

The system requirements for the School Management System include a minimum processor of Intel Core i5 or higher, 8GB of RAM (16GB recommended), and at least 30GB of free disk space. It is compatible with Windows , macOS, or Linux operating systems. The software requirements include the latest LTS version of Node.js, npm or yarn as a package manager, and a database such as MongoDB. A modern web browser like Google Chrome or Firefox is also required for optimal usage. Node.js version~14.6.0 or newer .

Backend Setup

This is the initial setup of the backend pay attention:

  1. Database Connection -
    1. Open you terminal
    2. npm install mongoose
    3. Establish a connection to a MongoDB database by configuring a connection URI and using Mongoose's built-in methods for database interaction
      1. Once connected, you can perform CRUD operations (Create, Read, Update, Delete) through models created from these schemas.


    1. Passport Configuration -
      1. First install the necessary dependencies, such as passport, passport-local, and express-session
      2. Passport is a middleware that simplifies user authentication
      3. Passport's middleware is integrated into the application with passport.initialize() and passport.session(), allowing for the seamless handling of login, logout, and session persistence.


Components

A model is created from a schema, which defines the structure and validation rules for documents in a collection, for example the assignement model is created from the assignement schema with key fields such as the assignment title, description, due date, assigned date, and the associated course.


Controllers handle the business logic for processing requests and interacting with the models, in a schoolSignin controller, the controller would handle the logic for managing a school's sign-in process, such as authenticating the user and initiating a session.


Middleware functions process incoming requests before they reach the route handlers.Used to manage cross-origin requests and controls which external domains are allowed to access resources on the server by setting policies on allowed origins, methods, and headers.


Routes define the API endpoints that handle different client requests. Routes are linked to specific HTTP methods (like GET, POST, PUT, DELETE) and direct these requests to controller functions. In a financeRoutes file, you might have routes for viewing balances (GET /balance), updating payments (PUT /payment/:id), and deleting transactions (DELETE /transaction/:id).


The server file configures the Express application, including middleware, routes, and port settings. It sets up the server to listen for incoming requests and respond to them appropriately. In the server.js file, you might configure middleware like express.json() to parse JSON request bodies and express.urlencoded() to parse URL-encoded request bodies.
